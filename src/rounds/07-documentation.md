# Round 7: 文档体系 (Documentation)

**Priority:** P1 (P0 for team/open-source projects)
**Role:** 开发者体验官
**Dependencies:** R1 (needs architecture understanding)
**Goal:** Establish documentation standards and fix critical gaps.

---

## What to Scan

```
- Read: README.md — what/why/how?
- Read: CLAUDE.md if exists
- Glob: CONTRIBUTING.md, CHANGELOG.md, SECURITY.md
- Read: 3-5 error messages — actionable?
```

## Key Questions

1. **CLAUDE.md** — Can Claude Code understand this project?
2. **README** — From clone to running in 5 steps?
3. **Contributor docs** — PR process known?
4. **Architecture docs** — System design without reading all code?
5. **Error messages** — What happened + why + how to fix?

## Standard Proposals

| Proposal | Description |
|:---------|:------------|
| CLAUDE.md creation | Project-specific conventions |
| README fixing | Prerequisites + setup + env vars + scripts |
| Contributing guide | Branch + PR + coding standards |
| Architecture doc | Module map + data flow |
| Error message standard | Format guideline |

## Output Requirements

Doc gap analysis, CLAUDE.md content, README checklist, onboarding flow.
